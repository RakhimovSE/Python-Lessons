Принципы программирования

- одна функция должна решать только ОДНУ конкретную задачу
- если функция выполняет несколько задач, надо разбить ее на несколько фукнций
- если функция возвращает какое-то значение, называть переменную, которую она возвращает, "result"
- использовать общепринятые названия переменных:
    * i - счетчик
    * j - счетчик 2-го уровня (для цикла внутри цикла)
    * arr - массив
    * n - кол-во элементов (обычно про массив)
    * m - кол-во столбцов в двумерном массиве (n тогда - кол-во строк)
    * st - строка

--------

Передача параметра в функцию по ссылке и по значению

По умолчанию в Python переменная передается по значению.

При создании переменной для нее выделяется ячейка памяти со своим определенным адресом, в нее записывается значение переменной.

ПЕРЕДАЧА ПО ЗНАЧЕНИЮ
Для каждого параметра создается новая ячейка памяти, значение переменной КОПИРУЕТСЯ из исходной ячейку в созданную, который передается в функцию. Если внутри функции меняется значение переменной, то исходная переменная остается прежней

Недостатки:
1. Необходимо выделять дополнительную память

--------

ПЕРЕДАЧА ПО ССЫЛКЕ
Параметр функции ССЫЛАЕТСЯ на исходную ячейку памяти. Если внутри функции меняется значение переменной, то исходная переменная также меняется.

КОРТЕЖ (tuple)

Кортеж - неизменяемый массив (задается один раз при создании, после этого нельзя менять ни количество элементов, ни сами элементы).
В питоне кортеж обозначается через круглые скобки
